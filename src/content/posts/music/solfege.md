你有没有遇到过这种情况：和朋友一起唱谱，明明看着同一份乐谱，唱出来的却不一样？

G 大调的音阶 G A B C D E F#，有人唱 do re mi fa sol la ti，而我唱的是 sol la ti do re mi fa。同一串音符，两种完全不同的唱法——这背后其实是两种不同的唱名体系：[固定唱名法与首调唱名法](https://en.wikipedia.org/wiki/Solf%C3%A8ge)。

## 唱名的起源

在聊两种唱名法之前，先说说 Do Re Mi 是怎么来的。

11 世纪的意大利，音乐理论家 [Guido of Arezzo](https://en.wikipedia.org/wiki/Guido_of_Arezzo) 从一首献给圣约翰的拉丁赞美诗 *[Ut queant laxis](https://en.wikipedia.org/wiki/Ut_queant_laxis)* 中提取了六个音节——每一行歌词的起始音刚好依次升高一个音级：

> **Ut** queant laxīs **re**sonāre fibrīs
> **Mi**ra gestōrum **fa**mulī tuōrum,
> **Sol**ve pollūtī **la**biī reātum,
> **S**ancte **I**ohannēs.
>
> 译：愿你的仆人，能放声歌颂，你奇妙的事迹，请洗去我们，污浊唇舌的罪过，圣约翰啊。

于是就有了 **Ut**（/yt/）、**Re**（/ɹeɪ/）、**Mi**（/miː/）、**Fa**（/fɑː/）、**Sol**（/soʊl/）、**La**（/lɑː/）六个唱名。后来在 17 世纪的意大利，Ut 因为不是开音节、不好唱，被改成了 **Do**（/doʊ/）。第七个音 **Si** 取自最后一行 **S**ancte **I**ohannēs 的首字母缩写。在英美体系中，19 世纪教育家 [Sarah Glover](https://en.wikipedia.org/wiki/Sarah_Glover) 将其改为 **Ti**（/tiː/），这样七个音节的首字母就各不相同了（而在法、意、俄等地区仍使用 Si）。

这就是我们今天熟悉的 **Do Re Mi Fa Sol La Ti / Si** 的由来。

## 首调唱名法（[Movable Do](https://en.wikipedia.org/wiki/Solf%C3%A8ge#Movable_do_solf%C3%A8ge)）

首调唱名法的核心思想是：**Do 永远是当前调的主音**。

在 C 大调中，C 唱 Do；在 G 大调中，G 唱 Do；在 D 大调中，D 唱 Do。无论什么调，大调音阶听起来永远是 Do Re Mi Fa Sol La Ti。

这种方式的好处是直觉——你不需要知道具体是哪个音，只需要感受音与音之间的关系。对于唱歌、即兴伴奏、流行音乐来说，这是非常自然的思维方式。国内的基础音乐教育中更常采用首调唱名法。在全球范围内，英语国家、中国、日本、匈牙利等地区广泛使用这种体系，其中匈牙利的 [Kodály 教学法](https://en.wikipedia.org/wiki/Kod%C3%A1ly_method) 就以首调唱名为核心。

## 固定唱名法（[Fixed Do](https://en.wikipedia.org/wiki/Solf%C3%A8ge#Fixed_do_solf%C3%A8ge)）

固定唱名法的核心思想是：**每个唱名对应一个固定的音**。

C = Do，D = Re，E = Mi，F = Fa，G = Sol，A = La，B = Ti（或 Si）——这个映射关系不随调性变化。在 G 大调中，G 唱 Sol 而不是 Do。

这种体系在意大利、法国、西班牙、俄罗斯等国家中是默认的——对他们来说，Do Re Mi 就是音的名字，就像英语里的 C D E 一样自然。一些专业音乐院校（如 [Juilliard](https://en.wikipedia.org/wiki/Juilliard_School)、[Curtis](https://en.wikipedia.org/wiki/Curtis_Institute)）也会在教学中使用固定唱名，但通常与首调唱名并行训练。

那么当 C 大调和 G 大调摆在一起时，两种唱名法的差异一目了然：

## 升降号怎么办？

自然音好说，但一旦出现升降号（♯ / ♭），两种唱名体系的处理方式就开始分化。

### 首调唱名法（Movable Do）的处理方式

在首调体系中，Do 表示的是“主音”，因此升降号本质上不是“音名变化”，而是**音级发生了改变**。

常见有两种处理方式：

#### 1. **直接加前缀**

在唱名前加“升”或“降”。例如：

- C# → 升 Do
- Bb → 降 Ti

实际演唱时仍然使用原音节，升降主要依赖读谱或听觉去判断。这种方式简单，但**无法精确表达音程变化**。

#### 2. **染色唱名法（Chromatic Solfège）**

为每个升降音分配独立的音节，通过元音变化来体现音级偏移——升号通常变为 **i**，降号变为 **e**：

| 自然音 | 升（♯） | 降（♭） |
| --- | ---- | ---- |
| Do  | Di   | —    |
| Re  | Ri   | Ra   |
| Mi  | —    | Me   |
| Fa  | Fi   | —    |
| Sol | Si   | Se   |
| La  | Li   | Le   |
| Ti  | —    | Te   |

这种体系在英美的视唱练耳训练中非常常见。相比“升/降前缀”，它的优势在于：

> **每一个音节都同时表达了“音级位置 + 升降变化”**

因此在涉及转调、半音进行、和声功能时更加清晰。

### 固定唱名法（Fixed Do）的处理方式

在固定唱名体系中，Do 始终等于 C，因此升降号就是**对具体音高的直接修改**，而不是音级变化。

常见做法是：

#### 1. **直接使用升降记号**

- C# → Do#（或直接读作 C#）
- Eb → Mi♭

这种方式与字母音名体系完全一致，重点在于**精确标识音高位置**。

#### 2. **较少使用染色唱名**

理论上也可以使用 Di / Me 等染色音节，但在固定唱名体系中并不常见。因为：

> 在固定唱名中，Do Re Mi 本身就是“音名”，而不是音级关系

引入染色唱名反而会混淆“音名”和“功能”的界限。

---

总结来看：

- 首调唱名：升降号 = **音级变化（功能变化）**
- 固定唱名：升降号 = **音高变化（物理位置变化）**

两者处理的是同一个现象，但表达的重点完全不同。

### 我的实际唱法

我自己用的是固定唱名，但实际唱的时候，升降号会被简化——直接归到最近的自然音唱名。例如：

- F# 我唱 Sol（归到上方的 G）
- Bb 我唱 Ti（归到 B）
- Eb 我唱 Mi（归到 E）

在我的体系里，**升降号不会改变基本唱名，而是以一种“偏移”的方式被感知**。

更准确地说，这并不是一种标准的唱名体系，而更像是一种基于绝对音感的“近似映射”——将 12 个音压缩到 7 个类别中，再由听觉去补偿细微差异。

### 一个有趣的现象

同样从 B 开始，不同的音阶我会唱出不同的结果：

| 音阶   | 音                  | 我的唱法                  |
| ---- | ------------------ | --------------------- |
| B 小调 | B C# D  E F# G  A  | Ti Do Re Mi Fa Sol La |
| B 大调 | B C# D# E F# G# A# | Do Re Mi Fa Sol La Ti |

同一个 B，小调里我唱 Ti，大调里我唱 Do。当整个音阶的音都升高了（D→D#、G→G#、A→A#），我的唱名框架似乎也随之整体“上移”。

这也侧面说明，我的听觉在一定程度上仍然受到“调性中心”的影响，而不是完全的固定映射。

## 两种唱名法的适用场景

|      | 首调唱名        | 固定唱名         |
| ---- | ----------- | ------------ |
| 核心逻辑 | Do = 主音     | Do = C       |
| 更适合  | 唱歌、即兴、流行/民乐 | 器乐演奏、视奏、音高定位 |
| 移调   | 天然友好，换调不换唱名 | 需要重新计算每个音    |
| 精确度  | 反映相对关系      | 反映绝对音高       |
| 常见于  | 基础音乐教育、合唱   | 专业训练、器乐学习    |

两种唱名法没有优劣之分，只是不同的思维方式。首调唱名让你专注于音乐的**结构**：什么是主音、什么是属音、和弦怎么走。固定唱名让你专注于音乐的**位置**：每个音在键盘上在哪、在谱子上在哪。

而像我这样“理论上用固定唱名、实际唱起来又不那么固定”的人，更像是不同听觉策略在脑中叠加的结果，而不是某一种标准体系的变体。

<script>
(function () {
  const SOLFEGE = ['Do','Re','Mi','Fa','Sol','La','Ti'];
  const COLORS = ['#e06c75','#e5c07b','#98c379','#56b6c2','#61afef','#c678dd','#be5046'];
  const SEMITONES = [0, 2, 4, 5, 7, 9, 11]; // major scale
  const ALL_NOTES = ['C','C#','D','D#','E','F','F#','G','G#','A','A#','B'];
  const WHITE_NAMES = ['C','D','E','F','G','A','B'];
  const WHITE_X = [10, 85, 160, 235, 310, 385, 460];
  const WHITE_CX = WHITE_X.map(x => x + 35);
  const BLACK_SEMI = [1, 3, 6, 8, 10];
  const BLACK_X = [60, 135, 285, 360, 435];
  const BLACK_CX = BLACK_X.map(x => x + 20);
  const BLACK_NAMES = ['C#','D#','F#','G#','A#'];

  function semiToPos(semi) {
    const bi = BLACK_SEMI.indexOf(semi);
    if (bi !== -1) return { cx: BLACK_CX[bi], isBlack: true, name: BLACK_NAMES[bi] };
    const wi = [0,2,4,5,7,9,11].indexOf(semi);
    return { cx: WHITE_CX[wi], isBlack: false, name: WHITE_NAMES[wi] };
  }

  function fixedSolfege(semi) {
    const map = [0, 0, 1, 1, 2, 3, 3, 4, 4, 5, 5, 6];
    return SOLFEGE[map[semi % 12]];
  }

  function buildSVG(el) {
    const root = el.dataset.root;
    const labels = el.dataset.labels;
    const title = el.dataset.title || '';
    const rootIdx = ALL_NOTES.indexOf(root);
    const scaleNotes = SEMITONES.map(s => (rootIdx + s) % 12);
    const isBoth = labels === 'both';
    const W = 540, KH = isBoth ? 130 : 140;
    const H = isBoth ? 240 : 220;

    let svg = `<svg viewBox="0 0 ${W} ${H}" xmlns="http://www.w3.org/2000/svg" style="max-width:${W}px;width:100%;font-family:system-ui,sans-serif;">`;

    // White keys
    for (let i = 0; i < 7; i++) {
      svg += `<rect x="${WHITE_X[i]}" y="10" width="70" height="${KH}" rx="4" fill="#fff" stroke="#333" stroke-width="1.5"/>`;
    }
    // Black keys
    for (let i = 0; i < 5; i++) {
      svg += `<rect x="${BLACK_X[i]}" y="10" width="40" height="${KH * 0.6}" rx="3" fill="#222" stroke="#000" stroke-width="1"/>`;
    }

    // Highlight scale notes
    scaleNotes.forEach(semi => {
      const p = semiToPos(semi);
      const cy = p.isBlack ? 10 + KH * 0.35 : KH - 15;
      const r = p.isBlack ? 10 : 12;
      svg += `<circle cx="${p.cx}" cy="${cy}" r="${r}" fill="#61afef" opacity="0.25"/>`;
    });

    // Note names on white keys
    for (let i = 0; i < 7; i++) {
      svg += `<text x="${WHITE_CX[i]}" y="${KH - 5}" text-anchor="middle" font-size="15" font-weight="bold" fill="#555">${WHITE_NAMES[i]}</text>`;
    }
    // Note names on black keys in scale
    scaleNotes.forEach(semi => {
      const p = semiToPos(semi);
      if (p.isBlack) {
        svg += `<text x="${p.cx}" y="${10 + KH * 0.35 + 5}" text-anchor="middle" font-size="11" font-weight="bold" fill="#eee">${p.name}</text>`;
      }
    });

    // Solfège labels
    const labelY1 = KH + 30;
    if (labels === 'unified') {
      scaleNotes.forEach((semi, i) => {
        const p = semiToPos(semi);
        svg += `<text x="${p.cx}" y="${labelY1}" text-anchor="middle" font-size="13" font-weight="bold" fill="${COLORS[i]}">${SOLFEGE[i]}</text>`;
      });
    } else if (isBoth) {
      // Movable Do row
      svg += `<text x="10" y="${labelY1}" font-size="12" fill="#e06c75" font-weight="bold">首调：</text>`;
      scaleNotes.forEach((semi, i) => {
        const p = semiToPos(semi);
        svg += `<text x="${p.cx}" y="${labelY1}" text-anchor="middle" font-size="13" font-weight="bold" fill="${COLORS[i]}">${SOLFEGE[i]}</text>`;
      });
      // Fixed Do row
      const labelY2 = labelY1 + 25;
      svg += `<text x="10" y="${labelY2}" font-size="12" fill="#61afef" font-weight="bold">固定：</text>`;
      scaleNotes.forEach(semi => {
        const p = semiToPos(semi);
        const fs = fixedSolfege(semi);
        const ci = SOLFEGE.indexOf(fs);
        svg += `<text x="${p.cx}" y="${labelY2}" text-anchor="middle" font-size="13" font-weight="bold" fill="${COLORS[ci]}">${fs}</text>`;
      });
    }

    // Title
    svg += `<text x="${W / 2}" y="${H - 25}" text-anchor="middle" font-size="12" fill="#888">${title}</text>`;
    if (isBoth) {
      svg += `<text x="${W / 2}" y="${H - 8}" text-anchor="middle" font-size="11" fill="#aaa">高亮的琴键为 ${root} 大调音阶中的音</text>`;
    }
    svg += `</svg>`;
    el.style.overflowX = 'auto';
    el.style.textAlign = 'center';
    el.innerHTML = svg;
  }

  document.querySelectorAll('.piano-kb').forEach(buildSVG);
})();
